# Markdown 笔记应用开发计划

## 1. 项目目标

实现一个完整的 Markdown 笔记应用，用户可以创建、编辑、预览、搜索和删除笔记。

应用采用左右分栏布局：

- 左侧：笔记列表、搜索框、标题输入、内容编辑器
- 右侧：Markdown 实时预览区域

数据保存在浏览器 `LocalStorage` 中，刷新页面后笔记不会丢失。

## 2. 技术选型

- 构建工具：Vite
- 前端框架：React
- 开发语言：TypeScript
- Markdown 渲染：react-markdown
- 代码高亮：react-syntax-highlighter
- 样式方案：Tailwind CSS
- 数据存储：LocalStorage

## 3. 核心功能

### 3.1 笔记创建

- 用户可以点击“新建笔记”创建一条空笔记。
- 新笔记需要包含唯一 ID、标题、内容、创建时间和更新时间。
- 新建后自动选中该笔记，方便立即编辑。

### 3.2 笔记编辑

- 用户可以输入笔记标题。
- 用户可以输入 Markdown 内容。
- 编辑内容时实时更新当前笔记。
- 修改后自动保存到 LocalStorage。

### 3.3 实时预览

- 右侧区域实时渲染 Markdown 内容。
- 支持常见 Markdown 语法：
  - 标题
  - 加粗
  - 列表
  - 引用
  - 链接
  - 代码块

### 3.4 代码高亮

- Markdown 代码块支持语法高亮。
- 使用 `react-syntax-highlighter` 渲染代码块。
- 根据代码块语言标识显示对应高亮效果。

### 3.5 笔记列表

- 左侧显示所有笔记。
- 每条笔记显示标题和更新时间。
- 点击列表项可以切换当前编辑的笔记。
- 当前选中的笔记需要有明显高亮状态。

### 3.6 搜索笔记

- 支持通过关键词搜索笔记。
- 搜索范围包括标题和内容。
- 搜索结果实时更新。

### 3.7 删除笔记

- 用户可以删除当前笔记。
- 删除前需要进行确认。
- 删除后自动选中剩余笔记中的第一条。
- 如果没有剩余笔记，则显示空状态。

## 4. 页面结构设计

### 4.1 整体布局

页面采用左右分栏：

- 左栏宽度固定，负责笔记管理和编辑。
- 右栏自适应宽度，负责 Markdown 预览。

### 4.2 左侧区域

左侧从上到下包含：

1. 应用标题
2. 新建笔记按钮
3. 搜索输入框
4. 笔记列表
5. 标题输入框
6. Markdown 内容编辑器
7. 删除按钮

### 4.3 右侧区域

右侧从上到下包含：

1. 预览标题
2. Markdown 渲染内容
3. 空内容提示

## 8. Git 学习路线

这个项目可以配合练习以下 Git 功能：

1. `git status`：查看当前工作区状态。
2. `git add`：把文件加入暂存区。
3. `git commit`：保存一个版本。
4. `git log`：查看提交历史。
5. `git diff`：查看文件修改内容。
6. `git branch`：创建和查看分支。
7. `git switch`：切换分支。
8. `git merge`：合并功能分支。
9. `git restore`：撤销工作区修改。
10. `git reset`：回退暂存区或提交。

## 9. 分支练习建议

建议使用下面的分支流程：

```bash
git switch -c feature/layout
git switch -c feature/note-editor
git switch -c feature/markdown-preview
git switch -c feature/search-delete
```

每个功能在独立分支开发，完成后合并回主分支。

## 10. 完成标准

项目完成时需要满足：

- 可以创建多条笔记。
- 可以编辑标题和内容。
- 可以实时预览 Markdown。
- Markdown 代码块有语法高亮。
- 可以搜索笔记。
- 可以删除笔记。
- 刷新页面后数据仍然存在。
- Git 历史中有清晰的多次提交记录。
